# Metric Filters для Nginx (ЛР8)

Стандартний **combined** access log Nginx містить код відповіді **перед** розміром відповіді. Приклад рядка:

`127.0.0.1 - - [17/Apr/2026:12:00:00 +0000] "GET / HTTP/1.1" 200 1234`

У CloudWatch **Metric filter** → **Space-delimited** можна задати поля; для простого варіанту використовуй **Pattern** (regex) у консолі.

## Приклади патернів (підлаштуй під реальний `log_format` у `nginx.conf`)

- **5xx**: підрахунок рядків, де після запиту йде статус 5xx  
  - Regex (приклад): `\" [5][0-9][0-9] `
- **4xx**: `\" [4][0-9][0-9] `

Після створення фільтра задай **Metric name** (наприклад `Nginx5xxCount`), **Value = 1** на збіг, **Default value** 0 за потреби.

> Якщо формат логу інший — відкрий кілька рядків `/var/log/nginx/access.log` на EC2 і піджени regex.
